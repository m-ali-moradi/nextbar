# Technologies

- **Java/Kotlin + Spring Boot**: Core backend framework for creating RESTful microservices.
- **Docker**: Used to containerize and manage service instances.
- **PostgreSQL**: Database for persisting orders, stock levels, and user roles.
- **Simulated Messaging**: For bottle collection and delivery confirmations.
- **UML Tools (UMLet, StarUML)**: Used for modeling domain and system architecture.

# Solution Architecture

Each microservice handles its own logic and database. Services communicate using REST APIs, with asynchronous simulations for delivery-related tasks. Admins access a central dashboard, while other users interact through role-specific views. Service orchestration ensures proper updates across the platform.
